# 旅行雷达 · 城市深度档案 起草基准规范（声音 + 深度 + schema）

> 目标：为缺档城市起草与现有 119 城**同一水准**的深度档案。真实、克制、有用。**宁缺勿造。**

## 一、声音（Voice）—— 必须内化
- 像**老练旅人给朋友的私房话**，不是旅游手册套话。简洁、有画面感、点到为止。
- 每条 highlights2 / mustEat2 写成 **"名称（一句精准点评/定位）"**，例：
  - `平安金融中心（116 层云际观光 · 全国第二高楼）`
  - `光明乳鸽（脆皮乳鸽 · 「天下第一鸽」）`
- **诚实红线（最高优先）**：只写**真实存在、可查证**的地点/店名/景点。**绝不编造**具体老字号、店名、数据。
  - 某类目本就稀少就**少写并如实**：年轻城市（如深圳）老字号极少；名山/自然单点没有老街小巷/老字号——就写少数真实的，或该 block 给 3-5 条真实的即可，**不要凑数硬编**。
  - 拿不准的排名/高度/开放状态等，用"约/据称"软化，并记进 caveats。
- 实用优先：radar 四件套给**真能用**的建议（怎么去、住哪、防坑、随俗）。

## 二、深度基准（按城市体量弹性，但 schema 字段齐全）
| block | 大城市/热门 | 中等/地级 | 小众·自然单点 |
|---|---|---|---|
| taglines | ~8-10 | ~6-8 | ~5-6 |
| highlights2 | 28-35 | 18-25 | 10-18 |
| mustEat2 | 22-28 | 15-22 | 8-15 |
| localFav | ~12 | ~8-10 | ~5-8 |
| oldShops | 尽真实所有 | 尽真实所有 | 真实少数或 3-5 条（无则宁少） |
| smallStreets | 10-12 | 8-10 | 3-6（自然地少写） |
| phenology | 12（真实物候花期） | 尽量 12 | 尽量 12，少也可 |
| wildlife | 5-7 | 5-7 | 5-7（自然地是重点） |
| seasons | **恰好 12**（每月一条 {month,icon,what}） | 恰好 12 | 恰好 12 |

## 三、schema（**逐字对齐参照城市**，键名/类型不得变）
起草前先 `Read` 打开 `_dev/data-src/deep.json`，看 **2-3 个同类参照城市**逐字对齐：
- 大都市→ `广州`；名山/自然→ `华山`/`五台山`/`张掖`；地级市→ `泉州`/`洛阳`/`绍兴`；小众→ `霞浦`/`松阳`；**国际城→ `巴黎`**（欧洲）/`首尔`/`新加坡`（亚洲）。
- 也开 `_dev/data-src/climate.json` 和 `budget.json` 看同一参照城的结构。

产出单个 JSON：
```
{
 "deep": {
   "taglines":[...], "highlights2":[...], "mustEat2":[...], "localFav":[...],
   "oldShops":[...], "smallStreets":[...],
   "phenology":[{"name","icon"(emoji),"when","where"}...],
   "wildlife":[{"name","icon","where","tip"}...],
   "seasons":[{"month":"1月".."12月","icon"(emoji),"what"} × 12],
   "radar":{
     "quickGlance":{"title","subtitle","bullets":[5-7],"tags":[数个]},
     "travel":{"fromTransport","cityTransport","stayZones":[{"name","vibe"}×3-4],"tips":[数条]},
     "tips":{"avoid":[4-6],"etiquette":[2-4]},
     "inspiration":{"locals":[4],"hidden":[4],"routes":[3]}
   }
 },
 "climate":{"1月":{"iconSet":[5 emoji],"condSet":[5 中文天气],"tempBase":[low,high],"tempVar":2}, ...12月},
 "budget":{"经济":{...},"中端":{...},"品质":{...}}   // 字段与参照城 budget 完全一致(stay/food/fun/trans/bigTrans/misc)，按当地物价调数
}
```

## 四、国际城额外
- `tips.etiquette` 含**签证/货币/礼俗/语言**要点；`travel` 含出入境/交通卡/时差提示；`inspiration.routes` 可含邻近连线。
- climate 用**当地真实气候**（南半球注意季节相反；地中海/热带/高纬各不同）。
- budget 数值按当地物价与汇率**量级合理**即可（非精确报价）。

## 五、交付与自检
- 写到 `scratchpad/cities/draft-<城市名>.json`（UTF-8）。
- **自检必过**：`seasons` 恰好 12；`climate` 恰好 12 月、每月 iconSet/condSet 各 5；deep 十个 block 键齐全；radar 四子键齐全；budget 三档字段与参照城一致。
- 另写出处/存疑到返回值 caveats（列需人工复核的点）。
