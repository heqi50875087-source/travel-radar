# 旅行雷达 · 城市深度档案补全任务书

> 本文写给负责搜集整理资料的 AI。请严格按第三节的 JSON 模板输出，交付物会被程序直接导入网站，**格式错一个引号都会导入失败**。

## 一、背景（30 秒读懂）

「旅行雷达」是一个面向独行旅行者的私人旅行网站，每座城市有一份 **14 区块深度档案**（必吃、老字号、本地人心水、老街小巷、物候历、防坑清单、住宿片区……）。现有 217 城中 **127 城还没有深度档案**（点开只有一张基础卡），你的任务就是为指定城市产出这份档案的数据。

网站的灵魂是**"本地人视角 + 独行者视角 + 说人话"**。参考现有档案的颗粒度：

- ✅ 好："富春茶社（光绪十一年创）· 得胜桥老店最正"、"灵隐门口的'野导游'和'假和尚'卖香带签，不理会"
- ❌ 差："当地有很多美食"、"注意防骗"（这种空话一条都不要）

## 二、你要做的事

1. 从第六节清单中**按批次**领取城市（建议一次做 5-8 城，质量优先于数量）
2. 对每座城市联网搜集资料（旅行攻略、本地媒体、点评、政府文旅站、知乎/小红书讨论均可作线索，**但每条信息要可信可查，拿不准的宁可不写**）
3. 按第三节模板输出 **一城一个 JSON 代码块**，代码块前只写一行城市名
4. 全部字段用**简体中文**；emoji 只用 1 个字符；不要在 JSON 里写注释

## 五、质量红线（违反任何一条整城退回）

1. **绝不编造**：店名、街名、年代、物候时间必须真实存在；找不到足够信息的字段，宁可少几条，不许凑数
2. **具体压倒一切**：每条都要有专名（店名/路名/村名/桥名），"当地特色美食"这类泛话= 0 分
3. **独行者视角**：一人食友好的小吃面点要充足；防坑和安全提示认真写；routes 里节奏别排成特种兵
4. **说人话**：像本地朋友发微信，不要百科腔、不要营销腔、不要"素有…之美誉"
5. **格式军规**：合法 JSON（双引号/无注释/无尾逗号）；seasons 和 climate 必须 12 个月齐全；iconSet 与 condSet 严格各 5 项；数字不带引号；month 键必须是 "1月" 到 "12月" 这种写法
6. 小城/纯景区（清单中标 △ 的）允许缩量：highlights2 ≥8、mustEat2 ≥6、oldShops/wildlife 可少至 0-3 条，其余照旧
